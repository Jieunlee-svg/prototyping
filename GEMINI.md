## UI/UX 컴포넌트 작성 규칙

### 1. 입력 폼 (Forms & Inputs)

- **플레이스홀더**: 모든 `input`에는 입력 형식을 보여주는 `placeholder`를 반드시 포함한다. (예: "휴대폰 번호 11자리를 입력하세요", "이메일 주소를 입력하세요")
- **자동 포커스**: 페이지 진입 시 첫 번째 입력 필드에 `autoFocus`를 설정한다.
- **실시간 유효성 검사**: 유효성 검사는 입력 중 또는 필드를 벗어날 때 즉시 시각적으로 표시한다.
- **버튼 활성화 로직**: 필수 항목이 모두 올바르게 입력되지 않으면 제출 버튼을 `disabled` 상태로 유지한다.
- **에러 메시지 위치**: 에러 메시지는 해당 입력창 바로 아래에 붉은색 텍스트로 표시한다.
- **에러 메시지 문구**: 사용자가 취해야 할 구체적인 행동으로 작성한다. (예: "올바른 이메일 형식이 아닙니다", "비밀번호는 8자 이상이어야 합니다")
- **비밀번호 보기**: 비밀번호 입력창에는 눈 아이콘을 배치하여 입력 내용을 확인할 수 있게 한다.
- **숫자 키패드**: 숫자 입력 필드에는 `inputMode="numeric"`을 설정하여 모바일에서 숫자 키패드가 뜨도록 한다.

### 2. 버튼 및 인터랙션 (Buttons & Interaction)

- **버튼 상태 시각화**: 버튼은 Hover, Active, Disabled 상태의 디자인이 명확히 구분되어야 한다.
- **로딩 상태**: 데이터 전송 중에는 버튼 내부에 스피너를 표시하고 중복 클릭을 방지한다.
- **파괴적 액션 경고**: '삭제' 등 되돌리기 힘든 작업은 버튼을 붉은색 계열로 하고, 클릭 시 확인 모달을 띄운다.
- **기본 동작 방지**: 모든 폼 제출 버튼은 `preventDefault` 처리하여 페이지가 새로고침되지 않도록 한다.

### 3. 알림 및 피드백 (Feedback & Notifications)

- **토스트 알림**: 성공적인 작업 후 화면 상단 또는 하단에 3초간 유지되는 토스트 메시지를 표시한다.
- **빈 상태(Empty State)**: 리스트에 데이터가 없을 경우 안내 메시지와 행동을 유도하는 버튼을 함께 표시한다.
- **스켈레톤 UI**: 데이터 로딩 중에는 레이아웃이 깨지지 않도록 스켈레톤 UI를 노출한다.
- **진행률 표시**: 파일 업로드 등 시간이 걸리는 작업은 프로그레스 바로 진행 상태를 전달한다.

### 4. 레이아웃 및 네비게이션 (Layout & Nav)

- **반응형 그리드**: 모든 레이아웃은 모바일, 태블릿, 데스크탑 환경에 맞춰 유연하게 변해야 한다.
- **상단 고정 헤더**: 주요 네비게이션은 스크롤 시에도 화면 상단에 고정(Sticky)되어야 한다.
- **뒤로가기 버튼**: 서브 페이지에는 항상 이전 페이지로 돌아갈 수 있는 명확한 뒤로가기 버튼을 배치한다.
- **페이지 제목**: 각 페이지 상단에 해당 페이지의 목적을 알 수 있는 H1 제목을 배치한다.
- **부드러운 스크롤**: 내부 링크 이동 시 `scroll-behavior: smooth`를 적용한다.
- **스크롤 최소화**: 정보가 많을 경우 가로 슬라이드, 탭 메뉴, 그리드 레이아웃을 활용하여 수직 스크롤을 최소화한다.

### 5. 타이포그래피 및 가독성 (Typography)

- **최소 폰트 크기**: 본문 폰트 크기는 최소 `16px` 이상을 유지한다.
- **줄 간격**: 본문 텍스트의 `line-height`는 `1.5~1.6`으로 설정한다.
- **대조비**: 배경색과 글자색은 충분한 명도 대비를 가져야 한다.
- **텍스트 생략**: 긴 텍스트가 영역을 벗어날 경우 말줄임표(`text-overflow: ellipsis`) 처리를 한다.

### 6. 접근성 및 기타 (Accessibility)

- **이미지 대체 텍스트**: 모든 `img` 태그에는 `alt` 속성을 반드시 작성한다.
- **엔터 키 지원**: 폼 입력 후 Enter 키를 누르면 제출 로직이 실행되어야 한다.
- **탭 키 탐색**: Tab 키 이동 순서는 시각적 배치 순서(왼쪽→오른쪽, 위→아래)와 일치해야 한다.
- **포커스 시각화**: Tab 키로 이동한 요소에 Focus Ring을 명확히 표시한다.
- **일관된 간격**: 요소 간 간격(margin, padding)은 4의 배수(4, 8, 12, 16px 등)를 사용한다.
